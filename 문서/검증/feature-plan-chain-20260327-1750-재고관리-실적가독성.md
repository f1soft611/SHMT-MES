# Feature Plan Chain Report

- 대상: 재고관리 컬럼 개선, 생산계획 대비 실적 가독성 개선
- 작성일시: 2026-03-27 17:50

## 1단계: 프로젝트 리더

## 요약

요구사항을 재고관리 화면 컬럼 정책 변경과 생산계획 대비 실적 화면 폰트 조정으로 분해했다.

## 변경/점검 범위

- 포함: frontend 화면 2개 파일 수정
- 제외: 신규 API, DB 스키마, MyBatis SQL, 권한 정책 변경

## Critical/High 이슈

- High: 재고관리 그리드에서 itemSeq와 itemNo 모두 품목코드로 보이면 사용자 혼동 위험

## 결정 사항

- itemSeq 헤더를 품목코드로 사용
- itemNo 헤더를 품목번호로 변경
- ERP 프로시저 미제공으로 itemName 컬럼 제거
- 폰트 조정은 ProdPlanResult 화면 한정

## 다음 단계 입력

백엔드 개발자용: ERP 프로시저 반환 컬럼을 기준으로 itemName 미제공을 확인하고, 프론트 컬럼 정책(품목코드/itemSeq, 품목번호/itemNo)을 확정하라.

## 2단계: 백엔드 개발자

## 요약

백엔드 검증 결과 ERP 프로시저는 itemName을 반환하지 않으며, 프론트는 itemSeq/itemNo 중심으로 표시해야 한다.

## 변경/점검 범위

- Controller/Service/DAO/MyBatis 결과 매핑 검증
- SQL 변경 필요성 검토

## Critical/High 이슈

- Critical/High 없음 (백엔드 변경 없음)

## 결정 사항

- API 계약 재사용 (itemSeq, itemNo)
- 백엔드 소스/매퍼 미수정 (ERP 프로시저 itemName 미제공 확인)
- 멀티 DB SQL 점검 포인트: SQL 미변경으로 영향 없음

## 다음 단계 입력

프론트 개발자용 API 계약: 재고조회 응답의 itemSeq/itemNo 필드를 사용해 그리드 컬럼을 품목코드/품목번호로 표시하고 itemName 컬럼은 제거하라.
