01 프로젝트 배경 및 과제

제조업의 주요 과제

  • ❌ 수작업 기반 생산계획 → 시간 소요, 오류 발생
  • ❌ 생산 현황 파악 지연 → 실시간 데이터 부족
  • ❌ ERP와 현장 데이터 단절 → 정보 동기화 지연
  • ❌ 높은 구축 비용 → 대형 MES 도입 어려움

SHMT-MES로 해결

  • ✅ 자동화된 생산계획 → 신속하고 정확한 계획
  • ✅ 실시간 현황 대시보드 → 즉시 파악
  • ✅ 영림원 ERP 자동 연동 → 데이터 동기화 자동화
  • ✅ 경량, 빠른 도입 → 3~6개월 내 운영

02 SHMT-MES 솔루션 개요

정의

영림원 ERP를 기반으로 하는 경량 MES → ERP 데이터를 활용한 실시간 생산 관리
→ 중소 제조 회사 최적화 (빠른 도입, 낮은 비용)
→ 다양한 환경 지원 (MSSQL, MySQL, Oracle 등)

3가지 핵심 가치

1️⃣ 영림원 ERP 기반

사원, 거래처, 생산의뢰 자동 동기화
기존 ERP 투자 활용 극대화

2️⃣ 경량하고 빠른 도입

3~6개월 내 운영 시작
중소 회사도 접근 가능한 비용

3️⃣ 확장 가능한 아키텍처

공정, 설비, 품목 자유로운 확장
향후 QMS, PLM 추가 용이

06 ERP 연동 인터페이스 (자동화)

영림원 ERP ↔ SHMT-MES 자동 동기화

📊 사원정보 동기화

출처: 영림원 ERP 사원 관리

내용: 사번, 이름, 부서, 직급

기능: 실시간 동기화 → 작업자 배치 & 권한 관리

📊 거래처정보 동기화

출처: 영림원 ERP 거래처 관리

내용: 거래처명, 주소, 담당자

기능: 구매/납품 거래처 관리 → 발주 연동

📊 생산의뢰 동기화

출처: 영림원 ERP 생산 의뢰

내용: 의뢰번호, 제품, 수량, 납기

기능: 자동 일정 수립 → 계획/실행/실적 추적

실행 방식

• 자동 실행: CRON 스케줄러 (일일/주일/월일 설정)
• 수동 실행: 관리자 화면에서 기간 선택 후 즉시 실행
• 에러 처리: 동기화 실패 시 로그 기록 + 알림
• 안전성: 동시성 제어로 중복 삽입 방지

07 기술 아키텍처

Backend: Spring Boot

  • ✓ Framework: eGovFrame 4.3.0
  • ✓ Language: Java 1.8
  • ✓ API: RESTful + JWT 토큰 인증
  • ✓ DB ORM: MyBatis (다중 DB 지원)
  • ✓ 스케줄러: Quartz + Spring Task

Database

  • ✓ Primary: MSSQL
  • ✓ 지원: MySQL, Oracle, Altibase, CUBRID, Tibero
  • ✓ 각 DB별 SQL 최적화

Frontend: React + TypeScript

  • ✓ React 19.x + TypeScript 4.9
  • ✓ UI: MUI 7 + Emotion
  • ✓ 상태관리: Context API + React Query 5
  • ✓ HTTP: Axios (단일 인스턴스)
  • ✓ 타입 안전성: any 사용 금지

보안 & 인증

  • ✓ JWT (Access: 60분, Refresh: 7일)
  • ✓ 권한: ADMIN, USER, VIEWER
  • ✓ @PreAuthorize 검증
